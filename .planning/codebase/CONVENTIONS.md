# Coding Conventions

## Naming
- **Components:** PascalCase (e.g., `PostCard.tsx`).
- **Functions/Variables:** camelCase (e.g., `createPost`).
- **Files:** camelCase for utilities, PascalCase for components.

## TypeScript
- Usage of `src/types/index.ts` for shared interfaces.
- Strict typing for API responses and library functions.

## Components
- Mark client-side interactivity with `'use client'`.
- Prefer functional components and hooks.
- Use Framer Motion for smooth transitions and interactions.

## Documentation
- Descriptive function names.
- Logical grouping of functions with comment dividers (e.g., `// ─── ... ───`).
